# 7天冲刺计划：把简历上的内容学会、讲清楚、做出来

## 总目标

7 天内，不追求把所有技术学成“资深工程师水平”，而是追求 4 件事：

1. 你能说清楚简历上每个技术为什么存在、解决什么问题。
2. 你能把两个个人项目从 0 到 1 做出来，并知道每个模块为什么这么设计。
3. 你能独立使用 Git 和 GitHub 管理项目、提交代码、写 README、展示项目。
4. 面试官追问时，你能从“原理、设计取舍、落地细节、踩坑”四个角度回答。

## 你的简历核心知识地图

你的简历真正需要掌握的，不是零散名词，而是 5 条主线：

1. 后端主线：`HTTP -> RESTful API -> FastAPI -> Pydantic -> JWT -> Swagger`
2. 数据主线：`MySQL -> 表设计 -> 索引 -> 事务 -> SQLAlchemy ORM -> Alembic`
3. 性能主线：`Redis -> 缓存命中 -> TTL -> 缓存一致性 -> 热点数据`
4. 部署主线：`Linux -> Docker -> Docker Compose -> Nginx -> 日志排查`
5. 工程主线：`Git -> GitHub -> 分支 -> 提交 -> README -> pytest -> 项目表达`

## 7 天安排

## Day 1：建立后端全景图 + Git/GitHub 入门

目标：

- 明白后端项目从请求到响应的完整链路
- 明白 FastAPI、MySQL、Redis、Docker、Nginx 各自扮演什么角色
- 学会 Git 最核心的 8 个命令
- 学会 GitHub 上仓库、分支、commit、pull request 的概念

产出：

- 你能口头讲清楚博客系统架构
- 你能自己看懂一个 FastAPI 项目的目录结构
- 你能把一个本地项目提交到 GitHub

## Day 2：FastAPI 核心原理 + 博客项目接口骨架

目标：

- 掌握路由、请求参数、响应模型、依赖注入、异常处理
- 明白 Pydantic 为什么比手写字典更安全
- 开始做博客 CMS 项目的接口骨架

产出：

- 用户、文章、分类、标签四类接口骨架
- 一套统一响应格式

## Day 3：MySQL + SQLAlchemy + Alembic

目标：

- 学会从业务需求反推表结构
- 理解一对多、多对多、索引、事务
- 掌握 ORM 映射和数据库迁移

产出：

- 博客项目数据库模型
- 迁移脚本
- 常见查询语句与 ORM 写法

## Day 4：JWT、权限、Redis 缓存、测试

目标：

- 理解认证和授权的区别
- 学会 JWT 登录态设计
- 学会 Redis 热点缓存和失效策略
- 学会 pytest 测正常、异常、边界

产出：

- 登录鉴权流程
- 热门文章缓存
- 基础测试用例

## Day 5：实时监控平台 + WebSocket + 告警规则

目标：

- 理解轮询和 WebSocket 的区别
- 做出设备实时监控平台的后端主流程
- 学会时序数据、告警、实时推送的基本设计

产出：

- 设备上报接口
- WebSocket 推送
- 告警入库逻辑

## Day 6：Docker、Compose、Nginx、Linux 部署

目标：

- 明白“代码能跑”和“服务可部署”之间差什么
- 掌握多容器协作
- 学会 Nginx 反向代理、静态资源代理、WebSocket 代理
- 学会日志排查和进程管理

产出：

- 两个项目的容器化部署版本
- 一份部署说明

## Day 7：面试表达 + 项目复盘 + GitHub 完善

目标：

- 把两个项目讲顺
- 准备高频追问答案
- 完善 GitHub README、接口说明、运行方式

产出：

- 面试问答稿
- 项目亮点表达模板
- GitHub 展示版仓库

## 项目策略

这 7 天我们不做“花哨项目”，只做和简历强绑定的 2 个项目：

1. 博客内容管理 API 系统
2. 实时设备监控与数据可视化平台

原因很简单：

- 它们已经在你简历上
- 做出来以后最适合面试追问
- 两个项目几乎覆盖你简历上全部技术点

## 这 7 天你最重要的学习方法

不是背定义，而是每学一个技术都回答这 4 个问题：

1. 它解决什么问题？
2. 不用它会怎样？
3. 为什么选它而不是别的方案？
4. 在我的项目里它具体放在哪？
